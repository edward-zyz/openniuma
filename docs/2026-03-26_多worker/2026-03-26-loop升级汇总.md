# spec: 2026-03-26 dev-loop.sh 升级汇总

除「多 Worker 并行」外，当日还有 6 项独立升级，按时间顺序整理如下。

---

## 1. 复杂度自动分流 + DESIGN_IMPLEMENT 合并模式

**Commits:** `ddad42d`, `7b9a548`

### 问题

基于 29 个功能的实际运行数据，发现三个问题：
- 复杂度全标"中"，人工标注形同虚设
- 4-session 完整流程（DESIGN → IMPLEMENT → VERIFY → MERGE）对中低复杂度任务开销过大
- VERIFY 100% 首次通过率，审查者没有对抗性

### 改动

**三级分流路径：**

| 复杂度 | 路径 | Sessions | 特点 |
|---|---|---|---|
| 低 | FAST_TRACK | 2 (实现+审查) | 无 spec/plan 文件 |
| 中 | DESIGN_IMPLEMENT | 3 (设计实现+审查+合并) | 精简 spec，无独立 plan |
| 高 | DESIGN → IMPLEMENT | 4 (设计+实现+审查+合并) | 完整 spec + plan + /brainstorming |

**复杂度由 Claude 自主判定：** 新任务默认进 DESIGN_IMPLEMENT，Claude 在会话开头快速浏览需求 + ≤5 个文件，按客观标准（文件改动数、是否有 DB migration、是否新增 API）决定降级 FAST_TRACK 或升级 DESIGN。

**去掉独立 plan 文件：** 中复杂度任务直接在 spec 中列出实现步骤，减少一次文件 I/O 和上下文切换。

## 2. Prompt 内联重构

**Commit:** `7b9a548` (+260/-152 行，当日最大改动)

### 问题

所有 phase 的 prompt 都是 `阅读 loop/autonomous-dev-loop.md 的 Phase X 部分`，导致 Claude 每个会话都要读 26600 tokens 的大文档，其中 ~98% 与当前 phase 无关。

### 改动

将所有 phase 的 prompt 从"引用外部文档"改为"直接内联精准指令"：

```
之前：阅读 loop/autonomous-dev-loop.md 的 Phase 1: DESIGN 部分...（~26600 tokens 上下文消耗）
之后：# DESIGN：高复杂度架构设计\n你是架构师。读取 loop-state.json...（~200-400 tokens）
```

同时提取公共片段为变量，避免重复：
- `COMMON_RULES` — 硬性红线（禁止合并 master、ESM 规范、门禁命令等）
- `COMMON_STATE_INSTRUCTIONS` — state 文件读写规范
- `COMMON_CHECKPOINT` — 断点续传逻辑

**效果：** 每个 Claude 会话节省 ~26000 tokens 上下文，同时消除 prompt 与文档 drift 的风险。

## 3. Worktree 路径迁移 + macOS 兼容性修复

**Commits:** `7b9a548`, `093a674`

### 问题

- Worktree 原来创建在 `../POI-loop-<slug>`（主仓库同级目录），多个 worktree 散落在父目录下，不好管理
- `ensure_worktree()` 被 `$()` 调用时，函数内的 `log()` 和 `git` 输出会污染 stdout 返回值，导致下游 `python3` 报 SyntaxError
- macOS 不自带 `stdbuf` 命令

### 改动

| 项目 | 之前 | 之后 |
|---|---|---|
| Worktree 父目录 | `$(dirname $MAIN_REPO_DIR)` (仓库同级) | `$MAIN_REPO_DIR/.trees/` (仓库内子目录) |
| Worktree 前缀 | `POI-loop` | `loop` |
| 完整路径示例 | `../POI-loop-keyword-search/` | `.trees/loop-keyword-search/` |

**stdout 泄漏修复：** `ensure_worktree()` 用 `exec 3>&2` 创建 fd3 指向 stderr，所有日志输出走 `>&3`，只有最后 `echo "$wt_path"` 走 stdout。

**macOS 兼容：** 移除 `stdbuf` 依赖。

## 4. State 文件自检与重建

**Commit:** `093a674` (+158 行)

### 问题

`loop-state.json` 被误删或丢失时，脚本静默退出，无法自恢复。

### 改动

新增 `ensure_state_file()` 函数，在脚本启动时自检：

```
启动 → loop-state.json 存在？
  ├── 是 → 继续
  └── 否 → 扫描 loop/tasks/*.md 重建
        ├── 解析 frontmatter（status, complexity, name, created_at）
        ├── status=done/closed/completed → 放入 completed
        ├── 其余 → status=pending
        ├── dev_branch = dev/backlog-batch-{today}
        └── current_item_id = 第一个 pending 任务
```

同时新增 **VERIFY 空 diff 回退**：进入 VERIFY/MERGE 前检查 `git log dev..feat --oneline`，如果 feat 分支没有新 commit（说明上一阶段实现失败），自动回退到 DESIGN_IMPLEMENT 并清空进度字段，避免空 diff 被判定 PASS。

## 5. Verbose 调试模式

**Commits:** `f800edb`, `19013cb`, `040c0fa`, `28c92a4`

### 问题

Loop 运行时 Claude 会话过程完全黑盒，只能从最终日志判断结果。`--output-format text` 有缓冲问题，终端长时间无输出。

### 改动

新增 `-v / --verbose` 参数：

```bash
./loop/dev-loop.sh -v              # 串行模式 verbose
./loop/dev-loop.sh --workers=3 -v  # 并行模式每个 worker 都 verbose
```

**实现方式：**

| 模式 | 输出格式 | 处理方式 |
|---|---|---|
| 普通 | `--output-format text` | 输出重定向到日志文件 |
| Verbose | `--output-format stream-json` | `tee` 保存原始 JSON + `jq` 实时解析显示 |

Verbose 模式 `jq` 实时解析 stream-json，提取关键信息：

```
💬 [文本输出内容]
🔧 tool_name: {input 前 200 字符}
✅ 完成 (123.4s, $0.56)
```

**附带修复：**
- `</dev/null` 显式关闭 stdin，消除 Claude CLI 的 3s 等待 warning
- `local` 关键字只能在函数内使用，while 循环中改为普通变量赋值

## 6. APFS Clone 加速 Worktree 依赖初始化

**Commit:** `05dc249`

### 问题

每个 worktree 创建后都执行 `npm install --prefer-offline`，即使依赖完全一致也要 ~60s。

### 改动

```
ensure_worktree() 依赖初始化流程：

主仓库 package-lock.json == worktree package-lock.json ？
  ├── 是 → cp -Rc node_modules（APFS CoW clone，~6s）
  │       ├── 根级 node_modules ✓
  │       ├── frontend/node_modules ✓
  │       ├── backend/node_modules ✓
  │       └── 验证：根 node_modules 不存在 → 回退 npm install
  └── 否 → npm install --prefer-offline（~60s）
```

**`cp -Rc`：** macOS APFS 的 Copy-on-Write 克隆，文件级引用复制，不占额外磁盘空间，速度从 ~60s 降到 ~6s。

## 变更影响矩阵

| 升级项 | 影响范围 | 风险 |
|---|---|---|
| 复杂度分流 | phase 状态机 + prompt | 低：向后兼容，现有 DESIGN/IMPLEMENT 路径不变 |
| Prompt 内联 | 所有 phase 的 prompt | 中：prompt 内容大改，但功能等价 |
| Worktree 路径 | 文件系统布局 | 低：旧 worktree 残留可手动清理 |
| State 自检 | 启动流程 | 低：仅在文件缺失时触发 |
| Verbose | Claude CLI 调用方式 | 低：默认关闭，不影响正常流程 |
| APFS Clone | npm install 替代方案 | 低：失败自动回退 |
